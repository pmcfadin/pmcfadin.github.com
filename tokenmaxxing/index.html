Tokenmaxxing

Stop burning tokens on work
CLI tools can do for free

Every file read, search, or edit stuffs full content into context — most of it noise. On a capped plan, that means hitting limits mid-task. The fix isn't a bigger plan; it's running a CLI tool first and handing the LLM only the result.

A single semantic search can consume 24,000+ tokens when an LLM reads the whole file unassisted. The same lookup with the right CLI tool: 188 tokens.

The pattern

Run a deterministic CLI tool first. The tool produces a compact, targeted result — a diff, a filtered file list, an exact passage. The LLM only ever sees that result, not the raw codebase. Token costs drop by 65–99% without changing the answer.
